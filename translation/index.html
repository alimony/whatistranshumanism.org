    How to translate

    Creating a new translation

    1. Create a new directory in the site root and name it with an IETF language code, for example sv-SE.
    2. Copy index.html from the site root to the new directory.
    3. Translate the variables in the top section of the new index.html:
      • title
      • description
      • localized_language_name
      • read_in_another_language
      • download_as_ebook
      • for_various_readers
      • for_kindle
    4. Translate the rest of the document, leaving all HTML as is. Internal links (starting with #) should be translated as well, and be based on the title of the linked question. These “slugs” are constructed by lowercasing the title, replacing all spaces with dashes, and converting all special character to ASCII. An easy way is to use an online slug generator, like this one.
    5. If you know your way around GitHub, push your translation to a new branch and open a pull request. More info on how this works is in the GitHub documentation.
    6. If not, just zip your new translation directory and send it to {{ site.email }}.

    Updating an existing translation

    On the top of this page there will be a list of translations that need updating, and a link to a diff showing what has changed since that language was last updated. With this information, you can make the necessary changes in the translation:

    1. Make sure you have the latest version of the website, by syncing your local copy with the main repository.
    2. Edit index.html in the language directory, updating it according to the latest version of the original.
    3. Open _config.yml and copy the value of latest_commit.
    4. Change the value of the based_on_commit variable in your translation by pasting the copied value from the previous step.
    5. Open a pull request for your update, or send it by email.

    As soon as your updated translation makes it to the live site, the update needed notice on the top of this page will go away.
